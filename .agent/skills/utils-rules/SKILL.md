# 工具函数模块编码规范

本 Skill 约束 `src/striker/utils/` 目录下的所有代码。utils 模块容纳无副作用的纯工具函数，被所有业务模块共享。

## 架构约束

- `utils/` 中的函数必须是**纯函数**：相同输入始终产生相同输出，无 I/O 副作用（网络、文件写入、全局状态修改）
- 允许的例外：`structlog` 日志调用（纯记录不影响返回值）
- 每个工具函数实现后**必须**向 `REGISTRY.md` 注册（AGENTS.md R06）
- 函数粒度：每个函数只做一件事，函数名必须自解释其功能
- 类型标注：所有参数和返回值必须有类型注解，通过 `mypy --strict`
- 坐标转换类函数必须处理单位转换（如度 ↔ 弧度、米 ↔ 毫米），禁止在上层业务代码中转换

### 依赖方向
- `utils/` 可依赖: `exceptions.py`, 标准库, 第三方库 (numpy 等)
- `utils/` 被依赖: **所有模块** — utils 是最底层的共享工具
- `utils/` 禁止依赖: `comms/`, `core/`, `flight/`, `safety/`, `payload/`, `vision/`, `config/`, `telemetry/`

### 数据流
- 输入: 原始数据（坐标、字符串、数值等）
- 输出: 转换/校验后的数据 — 无状态、无副作用

## 注册模式

| 注册项 | 说明 |
|--------|------|
| 坐标转换函数 | WGS84 ↔ 局部坐标系、度 ↔ 弧度等 |
| 数据校验函数 | GPS 校验、范围检查等 |
| 格式化函数 | 时间戳格式化、数值格式化等 |

注册示例:
```markdown
| deg_to_rad | 角度转弧度 | src/striker/utils/ | deg_to_rad(deg: float) -> float |
```

## 禁止模式

- **禁止**在 utils 函数中执行网络/文件 I/O — 纯函数原则
- **禁止**修改全局状态或模块级可变对象 — 函数必须无副作用
- **禁止**依赖业务模块 (comms, core, flight 等) — utils 是最底层，不能有向上的依赖
- **禁止**实现业务逻辑 — utils 只容纳通用工具函数，业务逻辑归对应模块
- **禁止**不注册新函数到 REGISTRY.md — 每个新增工具函数必须注册 (AGENTS.md R06)
